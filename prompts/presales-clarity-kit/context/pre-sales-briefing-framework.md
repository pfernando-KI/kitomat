# Pre-Sales Briefing Framework

## Zweck

Dieses Framework hilft, unklare Kundenanfragen vor einem Angebot zu strukturieren.

Die KI soll klären, was bereits bekannt ist, was fehlt und welche Rückfragen für den nächsten sinnvollen Schritt nötig sind.

## Klärungsdimensionen

| Dimension | Leitfrage |
|---|---|
| Anliegen | Worum bittet der Kunde offenbar? |
| Ziel | Was soll sich verbessern? |
| Ausgangslage | Was ist heute das Problem? |
| Zielgruppe | Für wen ist das Ergebnis gedacht? |
| Leistungsumfang | Welche Leistungen stehen im Raum? |
| Priorität | Was ist zuerst wichtig? |
| Zeitrahmen | Bis wann soll etwas passieren? |
| Budgetrahmen | Gibt es einen Rahmen? |
| Materialien | Was liegt schon vor? |
| Systeme | Welche Tools oder Prozesse sind betroffen? |
| Daten | Welche Datenarten sind betroffen? |
| Zuständigkeiten | Wer entscheidet und wer arbeitet mit? |
| Erfolgskriterien | Woran wird ein gutes Ergebnis erkannt? |

## Vorprüfung

Vor der eigentlichen Strukturierung sind vier Fragen zu prüfen:

| Prüffrage | Bedeutung |
|---|---|
| Passt die Anfrage zum Leistungsbereich? | Fachfremde Anfragen dürfen nicht wie passende Angebotsanfragen behandelt werden. |
| Sind personenbezogene, sensible oder vertrauliche Daten enthalten? | Solche Daten nicht wiederholen; bei hohem Risiko bereinigte Anfrage anfordern. |
| Liegt ein regulierter oder entscheidungsnaher Kontext vor? | HR, Gesundheit, Recht, Datenschutz, Compliance, Finanzen, Steuern und Bildung benötigen besondere Kontrolle. |
| Enthält die Anfrage zentrale Widersprüche? | Umfang, Zeit, Budget und Ziel dürfen nicht als stimmig ausgegeben werden, wenn sie es nicht sind. |

## Ampellogik

| Ampel | Bedeutung |
|---|---|
| Grün | Die Anfrage ist ausreichend klar für konkrete Angebotsvorbereitung mit fachlicher Kontrolle. |
| Gelb | Die Anfrage ist verständlich, aber wichtige Rückfragen sind nötig. |
| Rot | Die Anfrage ist zu unklar, fachfremd, widersprüchlich, sensibel oder riskant für Angebotsvorbereitung. |

## Grenzen

Die KI darf:

- Informationen strukturieren,
- Lücken markieren,
- Rückfragen formulieren,
- Erstgespräch vorbereiten,
- eine nicht-bindende Angebotsstruktur vorbereiten.

Die KI darf nicht:

- finale Angebote schreiben,
- Preise nennen,
- Aufwand verbindlich schätzen,
- Vertragsbedingungen formulieren,
- rechtliche, datenschutzrechtliche, steuerliche, finanzielle oder medizinische Beratung leisten,
- Entscheidungen über Menschen vorbereiten,
- Kundenzusage oder Ablehnung entscheiden,
- sensible Daten auswerten oder unnötig wiederholen.
