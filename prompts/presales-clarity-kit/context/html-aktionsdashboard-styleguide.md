# HTML-Aktionsdashboard Styleguide

## Zweck

Das HTML-Dashboard ist keine vollständige Analyse.

Es ist eine kompakte Arbeitsansicht für die bearbeitende Person.

Die Frage lautet:

> Was muss ich jetzt tun?

## Inhalt

Das Dashboard enthält nur:

- Ampelstatus,
- ein Satz zur Einordnung,
- maximal 4 offene Punkte,
- maximal 4 Rückfragen,
- genau eine nächste Aktion,
- kurzer Kontrollhinweis.

Das Dashboard enthält nicht:

- lange Analyse,
- vollständige Kundenanfrage,
- Kontextübersicht,
- Bewertungsmatrix,
- Quellenhinweise,
- lange Datenschutztexte,
- ausführliche Risikoabschnitte,
- allgemeine Erklärungen zur KI.

Bei roten Risikofällen soll das Dashboard bewusst bremsen. Es darf keine direkte Umsetzung nahelegen, wenn zuerst Daten bereinigt, Leistungsfit geklärt oder Human Review ausgelöst werden muss.

## Gestaltung

Das Dashboard soll:

- ruhig,
- hochwertig,
- klar,
- kompakt,
- gut lesbar,
- druckbar,
- auf Desktop und Mobile nutzbar sein.

## Optische Richtung

Empfohlene Farben:

| Zweck | Farbe |
|---|---|
| Hintergrund | `#f7f5f0` |
| Hauptfläche | `#ffffff` |
| Text | `#1f2933` |
| Sekundärtext | `#5f6875` |
| Akzent | `#1f7a75` |
| Weicher Akzent | `#e3eee9` |
| Gelb-Status | `#d9942b` |
| Rot-Status | `#b84a4a` |
| Linien | `#d8ddd6` |

## Layout

- Eine Hauptkarte mit klarer Kopfzeile.
- Darunter maximal zwei Spalten.
- Links offene Punkte.
- Rechts Rückfragen.
- Nächste Aktion als hervorgehobene Leiste.
- Kontrollhinweis klein im Fußbereich.
- Keine verspielten Animationen.
- Keine überladenen 3D-Effekte.
- Dezente Schatten sind erlaubt.

## Risikodarstellung

- Rot bedeutet: nicht angebotsreif oder nur mit deutlicher Einschränkung nutzbar.
- Bei sensiblen Daten keine Originaldaten im Dashboard wiederholen.
- Bei fachfremden Anfragen keine fachlichen To-dos zur fremden Leistung anzeigen.
- Bei regulierten oder entscheidungsnahen Fällen als nächste Aktion Human Review oder sichere Klärung nennen.

## Qualitätsregel

Das Dashboard ist gut, wenn die bearbeitende Person in unter 20 Sekunden erkennt:

- ob die Anfrage klar genug ist,
- welche Punkte zuerst fehlen,
- welche Fragen gestellt werden sollten,
- was der nächste Schritt ist.
